# OBS Floating Ball v0.3.0

本版本集中优化现有录制控制链路、最近录制管理与悬浮球显示效果，不扩展 OBS 场景、推流等功能。

## 主要调整

### 录制状态与自动隐藏

- 自动隐藏仅在真正进入录制状态时触发一次，状态轮询不会反复强制隐藏。
- 录制中手动显示悬浮球后，本次录制不再自动隐藏。
- 停止录制或录制中断开 OBS 后，曾被自动隐藏的悬浮球会自动恢复。
- 只有 OBS 确认真正进入录制或暂停状态后，才清除上一份录制的完成态与重命名路径。
- 悬浮条按钮与全局快捷键统一使用相同的开始录制状态逻辑。

### 最近录制与重命名

- 右键菜单新增“最近录制”子菜单，最多显示 8 个有效文件。
- 启动、打开菜单及点击条目时自动清理已经删除或移动的无效记录。
- 重命名成功后同步更新完成态路径和最近录制列表。
- 文件尚未写入完成时最多重试约 3 秒，等待提示只显示一次。
- 重命名窗口改为更紧凑的单行输入样式。

### 采集排除与悬浮球界面

- 悬浮条改为不透明实心胶囊，并使用 Windows `WDA_EXCLUDEFROMCAPTURE` 尝试从显示器采集中排除。
- 折叠或展开导致窗口尺寸变化后，会重新应用采集排除并将窗口限制在屏幕可用区域内。
- 状态球放大至 56px，右侧控制胶囊保持紧凑尺寸，减少圆形外轮廓的锯齿感。
- 使用超采样二值遮罩和白色边缘底色，尽量压低圆角锯齿并避免黑边。
- 受 Windows 二值窗口遮罩限制，圆形或圆角外缘仍可能存在轻微的 1 像素阶梯感。

## 安装与升级

1. 下载 `OBS.Floating.Ball.exe`。
2. 在 OBS 的“工具 → WebSocket 服务器设置”中启用 WebSocket，默认端口为 `4455`。
3. 运行悬浮球并填写连接密码；原有设置会继续保留。

## 验证

- Windows 10 19045、Python 3.11 环境构建。
- 自动化测试：49 项全部通过。
- 折叠和展开状态均已验证可成功应用采集排除。

## 兼容性

- 仅支持 Windows。
- 建议使用 OBS Studio 28 或更高版本（内置 obs-websocket 5.x）。
